## v4.1.1

This release improves the quality and consistency of generated release notes by tightening the guidance given to the model and sanitizing the model’s “semantic pass” output. The result is cleaner, more concise notes with fewer accidental references to internal details.

### Highlights

- Generated notes are cleaned up to remove echoed references, reduce duplication, and keep the final list short.
- Release-note generation now encourages terser, higher-signal output.

### For Non-Developers

Release notes produced by this package should read more cleanly and consistently. You’re less likely to see repetitive bullets or stray references that don’t belong in end-user notes, and the overall output should be shorter and easier to scan.

### Fixes

- Deduplicates semantic notes and limits the final set to a small number of high-signal items.
- Sanitizes semantic-pass output to remove echoed references and normalize formatting.

### Performance

- Reduces downstream processing and review effort by capping the number of semantic notes returned.
